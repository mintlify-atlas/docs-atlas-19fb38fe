## Overview

The NoOCR frontend is a modern React application built with TypeScript, Vite, and Tailwind CSS. It provides a clean interface for uploading documents, managing cases, and performing visual search.

## Core Components

### Search Component

The search interface handles semantic search and visual QA.

**Source:** `no-ocr-ui/src/components/Search.tsx`

```typescript
type SearchResult = {
  image_base64: string;
  pdf_name: string;
  pdf_page: number;
  score: number;
};

export default function Search() {
  const { user } = useAuthStore();
  const [selectedCase, setSelectedCase] = useState('');
  const [searchQuery, setSearchQuery] = useState('');
  const [results, setResults] = useState<SearchResult[]>([]);
  const [isSearching, setIsSearching] = useState(false);
  const [answers, setAnswers] = useState<{ [key: number]: { answer: string } }>({});
  const [cases, setCases] = useState<Case[]>([]);
  
  // Load cases on mount
  useEffect(() => {
    async function fetchCases() {
      const response = await fetch(`${noOcrApiUrl}/get_cases?user_id=${user?.id}`);
      const data = await response.json();
      setCases(data.cases || []);
    }
    fetchCases();
  }, [user]);
  
  // Handle search submission
  const handleSearch = async (e: React.FormEvent) => {
    e.preventDefault();
    if (!selectedCase || !searchQuery) return;
    
    setIsSearching(true);
    setResults([]);
    setAnswers({});
    
    try {
      // Call search API
      const response = await fetch(`${noOcrApiUrl}/search`, {
        method: 'POST',
        headers: { 'Accept': 'application/json' },
        body: new URLSearchParams({
          user_query: searchQuery,
          user_id: user?.id || '',
          case_name: selectedCase,
        }),
      });
      
      const data = await response.json();
      setResults(data.search_results || []);
      
      // Fetch answers for each result
      data.search_results.forEach((result: SearchResult, index: number) => {
        fetchAnswer(searchQuery, selectedCase, result.pdf_name, result.pdf_page)
          .then(answer => {
            setAnswers(prevAnswers => ({ ...prevAnswers, [index]: answer }));
          });
      });
    } catch (error) {
      console.error('Search failed:', error);
    } finally {
      setIsSearching(false);
    }
  };
  
  // Fetch VLM answer for a specific page
  const fetchAnswer = async (
    userQuery: string,
    caseName: string,
    pdfName: string,
    pdfPage: number
  ) => {
    try {
      const response = await fetch(`${noOcrApiUrl}/vllm_call`, {
        method: 'POST',
        headers: { 'Accept': 'application/json' },
        body: new URLSearchParams({
          user_query: userQuery,
          user_id: user?.id || '',
          case_name: caseName,
          pdf_name: pdfName,
          pdf_page: pdfPage.toString(),
        }),
      });
      
      const data = await response.json();
      return data;
    } catch (error) {
      console.error('Fetching answer failed:', error);
      return { answer: 'NA' };
    }
  };
  
  // Render search form and results...
}
```

**Key Features:**
- **Case Selection**: Dropdown to select which case to search
- **Search Form**: Text input for semantic queries
- **Async Results**: Search results appear first, then VLM answers load progressively
- **Image Display**: Base64-encoded images displayed with metadata
- **Modal View**: Click to enlarge images

## State Management (Zustand)

The app uses Zustand for lightweight state management.

**Auth Store Example:**

```typescript
import { create } from 'zustand';
import { User } from '@supabase/supabase-js';

interface AuthState {
  user: User | null;
  setUser: (user: User | null) => void;
}

export const useAuthStore = create<AuthState>((set) => ({
  user: null,
  setUser: (user) => set({ user }),
}));
```

**Usage in Components:**

```typescript
import { useAuthStore } from '../stores/authStore';

function MyComponent() {
  const { user, setUser } = useAuthStore();
  
  // Access user.id for API calls
  // Call setUser(newUser) to update state
}
```

### API Call Pattern

All API calls use the Fetch API with FormData for multipart uploads:

```typescript

    user_id: user?.id || '',

```

## Authentication (Supabase)

The app uses Supabase for user authentication.

**Setup:**

```typescript
import { createClient } from '@supabase/supabase-js';

const supabaseUrl = import.meta.env.VITE_SUPABASE_URL;
const supabaseAnonKey = import.meta.env.VITE_SUPABASE_ANON_KEY;

export const supabase = createClient(supabaseUrl, supabaseAnonKey);
```

**Auth Components:**

- `LoginForm.tsx`: Email/password login
- `RegisterForm.tsx`: New user registration
- `LogoutButton.tsx`: Sign out functionality
- `AuthGuard.tsx`: Protected route wrapper

**Example: Protected Route**

```typescript
import { Navigate } from 'react-router-dom';
import { useAuthStore } from '../../stores/authStore';

export function AuthGuard({ children }) {
  const { user } = useAuthStore();
  
  if (!user) {
    return <Navigate to="/login" />;
  }
  
  return children;
}
```

## Development Workflow

### Running the Dev Server

```bash
cd no-ocr-ui
npm install
npm run dev
```

Vite will start the dev server at `http://localhost:5173` with:
- Hot module replacement (HMR)
- Fast refresh for React
- TypeScript type checking

### Building for Production

```bash
npm run build
```

Produces optimized static files in `dist/`:
- Minified JavaScript bundles
- Optimized CSS
- Asset optimization
- Tree-shaking for smaller bundle size

### Linting

```bash
npm run lint
```

Runs ESLint with React-specific rules.

## Performance Optimizations

### Code Splitting

Vite automatically code-splits by route, loading only necessary JavaScript for each page.

### Image Optimization

- Backend resizes images to 512x512 before sending
- JPEG format with progressive encoding
- Base64 encoding for inline display

### Async Answer Loading

Search results appear immediately, while VLM answers load progressively:

```typescript
// Show results first
setResults(data.search_results || []);

// Then fetch answers asynchronously
data.search_results.forEach((result: SearchResult, index: number) => {
  fetchAnswer(searchQuery, selectedCase, result.pdf_name, result.pdf_page)
    .then(answer => {
      setAnswers(prevAnswers => ({ ...prevAnswers, [index]: answer }));
    });
});
```

This provides instant feedback while answers generate.
